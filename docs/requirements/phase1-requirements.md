# 河南省生态环境厅环境监测工勤技能考试系统 - 商用级改造（阶段一）

## Summary

对考试系统进行全方位商用级改造：重新设计全部前端页面（首页、刷题配置、考试配置、答题、结果、成绩统计、答案解析、错题本等），采用科技感深蓝/绿色调的专业政务风格；同时优化后端代码结构，提升系统稳定性与可维护性，强化错误处理与缓存机制。阶段一聚焦电脑端网页改造，验收通过后再启动微信小程序改造。

---

## Problem Frame

当前系统基于轻量级原型开发，已具备基本功能（刷题、考试、成绩统计、错题本），但视觉风格与交互体验较为朴素。前端使用内联CSS，缺少系统性的设计规范；后端代码结构有待优化，错误处理与缓存机制不够完善。作为政务/教育类考试系统，需要提升专业形象与用户体验，同时确保系统稳定可靠，满足商用级别的质量要求。

---

## Key Flows

- F1. 用户访问首页 → 查看题库统计 → 选择功能（刷题/考试/成绩/错题）→ 进入对应页面
- F2. 刷题流程：选择类型/分类/题数 → 开始练习 → 答题（实时反馈对错）→ 完成查看结果 → 可重做错题
- F3. 考试流程：选择题数/时长 → 生成试卷 → 答题（计时）→ 提交 → 查看成绩与答案解析
- F4. 错题管理：查看错题列表 → 选择题目重新练习 → 巩固薄弱知识点

## Requirements

**[前端 - 视觉风格]**

- R1. 采用科技感深蓝/绿色调配色方案，主色为 #1b5e20 ~ #2e7d32 ~ #43a047 绿色渐变系
- R2. 背景色使用浅灰色 #f0f2f5，卡片背景使用白色 #ffffff
- R3. 错误/警示使用 #f44336 红色，正确使用 #4caf50 绿色
- R4. 文字色：主文字 #333，次要文字 #666
- R5. 所有页面保持风格统一，共享统一的配色和组件样式

**[前端 - 页面改造]**

- R6. 首页：顶部深绿渐变banner，统计卡片网格布局，功能入口卡片悬停效果
- R7. 刷题配置页：表单样式统一，下拉选择框风格一致，开始/返回按钮样式规范
- R8. 考试配置页：试卷题数/时长选择，加载状态提示，提交按钮样式
- R9. 答题页：进度条、计时器、题目卡片、选项列表、答题卡（考试模式）
- R10. 结果页：大字体分数展示、通过/未通过状态、动作按钮组
- R11. 成绩统计页：统计数据卡片、历史记录列表、时间/分数/题数显示
- R12. 答案解析页：逐题展示、正确答案高亮、用户答案对比
- R13. 错题本页：错题列表、重新练习按钮、空状态提示

**[前端 - 交互优化]**

- R14. 选项点击后即时高亮显示已选答案
- R15. 练习模式：答题后立即显示对错反馈，正确绿色边框，错误红色边框
- R16. 考试模式：答题卡显示已答/未答状态，点击可跳转
- R17. 计时器：最后5分钟黄色警告，时间到自动交卷
- R18. 交卷确认：显示未答题目数量，防止误提交

**[前端 - 响应式设计]**

- R19. 桌面端（>900px）：多列布局，舒适间距
- R20. 平板/手机端（≤600px）：单列布局，按钮适应屏幕宽度

**[前端 - 组件规范]**

- R21. 按钮：圆角6px，hover状态有轻微上移和颜色加深效果
- R22. 卡片：圆角12px，阴影 0 2px 8px rgba(0,0,0,0.08)
- R23. 输入框/下拉框：圆角6px，focus状态边框变绿
- R24. 进度条：圆角4px，绿色渐变填充

**[后端 - 代码结构]**

- R25. 后端代码按功能模块组织（数据加载、路由、工具函数）
- R26. 统一的错误处理格式 { error: '错误描述' }
- R27. 关键操作添加console.log日志（启动、API请求、错误）
- R28. 常量配置集中管理（端口、文件路径、限制数量）

**[后端 - 稳定性]**

- R29. 所有文件读取操作使用try-catch包裹，失败时使用默认值
- R30. 请求参数验证：必要参数缺失返回400状态码
- R31. 题目/记录不存在时返回404状态码
- R32. 服务器内部错误返回500状态码

**[后端 - 缓存优化]**

- R33. recentlyUsedIds数组限制最大200条，超出后删除最老的记录
- R34. 历史记录限制最大200条，超出后裁剪
- R35. 数据文件不存在时自动创建空文件

## Acceptance Examples

- AE1. **Covers R6, R7.** 打开首页看到深绿渐变banner、4个统计卡片（总题数/判断题/选择题/错题数）、4个功能入口卡片，风格统一专业

- AE3. **Covers R16, R17.** 考试模式下，底部答题卡显示已答（红色）和未答（蓝色）状态，计时器最后5分钟变为黄色警告
- AE4. **Covers R29, R31.** 请求不存在的题目ID时，返回404 JSON错误 { error: '题目不存在' }

## Success Criteria

- **用户体验：** 视觉风格专业统一，交互流畅自然，符合政务/教育系统定位
- **功能正确性：** 刷题、考试、交卷、评分、结果展示全流程正确无误
- **系统稳定性：** API响应正常，错误处理完善，无崩溃或异常
- **代码质量：** 结构清晰，易于维护，日志完备

## Scope Boundaries

### Deferred for later

- 用户注册登录系统（当前localStorage匿名机制已可满足基本需求）
- 微信小程序改造（阶段一验收后启动）
- 高级数据分析功能（正确率趋势、知识点掌握度等）
- 题目收藏/笔记功能

### Outside this product's identity

- 引入数据库（继续使用JSON文件，简化运维）
- 微服务架构（保持轻量高效的单一服务器架构）

---

## Key Decisions

- **保持轻量架构：** 后端不进行大规模重构，继续使用Express+JSON文件，确保快速迭代和简单运维
- **前后端分离部署：** 前端作为静态资源由Express服务，后端提供RESTful API
- **API兼容优先：** 后端优化时不改变API接口格式，确保现有前端和微信小程序无需修改即可继续使用

## Dependencies / Assumptions

- 端口：默认3000，冲突时可通过环境变量PORT修改
- 题库数据：questions.json格式正确，包含2850道题目
