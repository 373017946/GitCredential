# 考试系统商用级别升级

## Summary

将河南省生态环境厅环境监测工勤技能考试系统从现有版本升级为商用级别产品：政府风格 UI 全面重构（深蓝+金色配色）、后端架构优化（数据库、缓存、安全）、多用户微信授权登录、小程序+网页端双端支持。

---

## Problem Frame

当前系统已在网页端和手机端可用，但存在以下商用级别差距：
1. **UI 形象** - 现有绿色主题偏消费类，不符合政府/事业单位的严肃专业形象
2. **用户体系** - 匿名用户模式，多人使用数据混在一起，无法独立统计
3. **后端架构** - JSON 文件存储无法支撑商用并发和数据一致性
4. **部署扩展** - 当前仅支持单节点部署，小程序需要适配

---

## Actors

- A1. **考生** - 通过微信授权登录，独立使用刷题、考试、错题本功能
- A2. **系统管理员** - （不在本项目范围内）管理题库、查看数据统计

---

## Key Flows

- F1. **微信授权登录**
  - **Trigger:** 用户打开系统
  - **Steps:**
    1. 系统检测到未登录，展示微信登录按钮
    2. 用户点击按钮，调用微信 OAuth2 授权
    3. 授权成功后获取 openid，用户信息入库
    4. 系统创建/更新用户 session，展示首页
  - **Outcome:** 用户以独立身份使用系统，历史记录和错题本独立

- F2. **在线刷题**
  - **Trigger:** 用户选择题目类型和数量
  - **Steps:**
    1. 用户配置刷题参数（类型、分类、数量）
    2. 系统返回随机题目（含答案）
    3. 用户答题，实时显示对错和反馈
    4. 完成后保存练习记录到用户历史
  - **Outcome:** 用户完成练习，错题自动收录

- F3. **模拟考试**
  - **Trigger:** 用户发起考试
  - **Steps:**
    1. 用户配置考试参数（题数、时长）
    2. 系统生成不含答案的试卷
    3. 用户在计时器内作答，可切换答题卡
    4. 交卷后系统评分并保存成绩
  - **Outcome:** 用户获得考试成绩和排名区间

- F4. **错题本**
  - **Trigger:** 用户想复习错题
  - **Steps:**
    1. 系统汇总用户历史记录中的错题
    2. 用户选择全部重练或选择性重练
    3. 系统加载错题供用户练习
  - **Outcome:** 用户针对性地强化练习

## Requirements

### R1. 政府风格 UI

- R1.1. 主题配色：主色深蓝 `#1a3a6b`，辅助色金色 `#c9a84c`，强调色白色，背景 `#f5f6f8`
- R1.2. 标题字体：微软雅黑 / 思源黑体；正文：系统默认无衬线
- R1.3. 布局：顶部深蓝横条导航，卡片式内容区域，圆角 6px，边距统一
- R1.4. 响应式：支持 375px~1920px 宽度，手机/平板/桌面自适应
- R1.5. 图标：使用 SVG 或 Unicode 符号，保持简洁专业
- R1.6. 首页：4 功能卡片（在线刷题 / 模拟考试 / 成绩统计 / 错题本），统计数字展示区
- R1.7. 题目页：题目内容区 + 选项区 + 答题卡（考试模式），进度条 + 计时器（考试模式）
- R1.8. 结果页：分数大字展示 + 及格/不及格状态 + 正确/错误/未答统计

### R2. 后端架构优化

- R2.1. 数据库：SQLite（文件数据库，支持高并发，比 JSON 更适合商用）
- R2.2. 缓存层：内存缓存热点数据（最近题目、用户 session），减少数据库查询
- R2.3. 用户表：id, openid, nickname, avatar, created_at, last_login
- R2.4. 历史记录表：id, user_id, score, correct_count, wrong_count, total_count, duration, is_exam, wrong_question_ids, created_at
- R2.5. 题目表：id, code, type, category, subject, content, options(JSON), answer, source
- R2.6. API 规范化：统一响应格式 `{code, message, data}`，错误码体系
- R2.7. 接入层：express + mysql2/sqlite3，数据层使用 ORM 或 raw sql
- R2.8. 日志：请求日志 + 错误日志，支持分级（info/warn/error）
- R2.9. 配置文件：环境变量配置（PORT, 数据库路径, JWT_SECRET 等）

### R3. 多用户微信登录

- R3.1. 微信 OAuth2 授权登录（公众平台网页授权）
- R3.2. 用户 openid 作为唯一标识（不重复注册）
- R3.3. 新用户首次登录自动创建账号
- R3.4. Session 管理：JWT token，token 有效期 7 天
- R3.5. 7天免登录：登录状态存储7天，刷新 token 无需重新授权
- R3.6. 登录状态持久化：token 存储在客户端（localStorage/小程序 storage）
- R3.7. 小程序端：使用 `wx.login()` 获取 code，换取 openid

### R4. 小程序适配

- R4.1. WebView 嵌入：将现有 H5 页面通过 web-view 组件在小程序中运行
- R4.2. 独立登录页：小程序内独立的登录页面，使用 `wx.login()` 获取 code，换取 openid
- R4.3. 7天免登录：登录状态存储7天，刷新 token 无需重新授权
- R4.4. API 适配层：小程序端的 request 网络请求需要适配（域名白名单、HTTPS）
- R4.5. 页面标题：动态设置小程序顶栏标题
- R4.6. 分享功能：支持小程序内分享页面

### R5. 考试核心功能

- R5.1. 刷题模式：按类型/分类/数量随机出题，返回答案，实时反馈
- R5.2. 考试模式：35%判断题+65%选择题，计时，交卷后出分
- R5.3. 错题本：汇总历史错题，支持重新练习
- R5.4. 成绩统计：平均分、及格率、考试/练习次数
- R5.5. 历史记录：分页展示历史成绩，可查看详情

### R6. 安全与性能

- R6.1. HTTPS：生产环境强制 HTTPS（小程序要求）
- R6.2. 跨域：配置 CORS 白名单
- R6.3. SQL 注入防护：参数化查询
- R6.4. 请求限流：同一 IP 限流（防止刷题 API 被滥用）
- R6.5. 题目缓存：内存缓存已加载题目，减少数据库查询

## Acceptance Examples

- AE1. **Covers R1.1, R1.6.** Given 用户打开首页，when 页面加载完成，then 深蓝+金色主题、4 功能卡片、统计数据正常展示。
- AE2. **Covers R3.1, R3.3.** Given 新用户打开系统，when 点击微信登录，then 授权后自动创建账号并进入首页，历史记录为空。
- AE3. **Covers R2.1, R2.3, R2.4.** Given 用户完成一次考试，when 提交试卷，then 成绩正确保存到数据库，历史记录可查询。
- AE4. **Covers R4.1, R4.3.** Given 用户在微信小程序中打开，when 加载系统，then H5 页面正常显示，微信登录流程完成。
- AE5. **Covers R5.2.** Given 用户参加50题模拟考试，when 考试计时60分钟到，then 自动交卷并显示成绩。

## Success Criteria

- 网页端和微信小程序端均可正常使用，界面风格统一
- 用户登录后数据完全独立（历史记录、错题本、成绩统计）
- 后端可支撑 100+ 并发用户
- API 响应时间 < 200ms（题库查询）
- 小程序可通过微信审核（功能完整、无违规）

---

## Key Decisions

- **SQLite over MySQL**：开发/部署简单，单文件数据库适合中小企业，无需额外部署 MySQL 服务
- **H5 + 小程序 web-view**：复用现有前端代码，小程序只做容器，减少维护成本
- **JWT over Session**：无状态更适合分布式部署，JWT token 可自包含用户信息
